# 实时查询界面

本项目是一个独立的全栈查询界面。浏览器只请求本项目的后端，再由后端读取已配置的上游接口；页面不使用种子数据、浏览器存储、JSON 导入或本地持久化。

## 运行

```bash
npm install
npm run dev
```

浏览器打开终端显示的本地地址。

## 页面

- **跟单大厅**：连红 / 胜率 / 盈利三张榜单，以及「人气跟单 / 跟单总额」两种排序的进行中方案列表。
- **发起人主页**：累计奖金、7 日盈利、7 日命中、近 5 场战绩，以及该发起人的历史发单。
- **神单详情**：方案对阵表（场次 / 主队客队 / 投注内容 / 赛果）与「跟单次数」下的跟单用户列表。

支持地址栏 `#profile/<发起人id>`、`#order/<订单id>`、`#order/<订单id>/buyers` 直达。

## 数据显示

- 大厅、搜索、资料页和订单详情均在当前页面会话中读取后端响应。
- 头像进入资料页时会立即查询发起人资料和历史订单；响应未提供的字段显示为“暂无返回”，不会用旧数据补全。
- 订单编码只显示接口实际返回的字段：`betCode` 为空但 `betCodeForResult` 存在时以后者为准；已确认的编码映射为可读玩法（如比分 `11` → `1:1`、半全场 `4` → `平-平`），没有映射依据的保留原编码，不猜测投注选项。
